## 개요

SOOL의 분석(Analytics)은  
단순한 조회 수 집계가 아니라  
**전통주 데이터 이해 → 사용자 반응 → 서비스 확장**을 위한
의사결정 도구로 설계되었다.

초기 단계에서는 **기본 통계 중심**,  
확장 단계에서는 **사용자 행동 기반 분석**으로 발전시킨다.

---

## 분석 설계 원칙

- 지표는 많을수록 좋은 것이 아니다
- “행동을 바꿀 수 있는 지표”만 추적
- MVP 단계에서는 **서버 부담이 없는 구조** 우선

---

## Phase 1: 데이터 자체 분석 (현재)

### 1️⃣ 전통주 분포 분석

#### 주요 지표
- 주종별 분포 (막걸리 / 증류주 / 약주 등)
- 지역별 분포
- 도수 분포 (Histogram)

#### 목적
- 데이터 편향 확인
- 추천 알고리즘 설계 기초 자료

---

### 2️⃣ 도수 통계

| 지표 | 설명 |
|---|---|
| 평균 도수 | 전체 전통주 평균 |
| 중앙값 | 분포 왜곡 확인 |
| 최소 / 최대 | 극단값 확인 |

> 도수 기반 필터링 및 추천 기준으로 활용

---

### 3️⃣ 지역별 집계

- 지역별 전통주 개수
- 특정 지역 집중 여부

> 지역 기반 큐레이션 콘텐츠 확장 가능성 판단

---

## Phase 2: 사용자 반응 분석 (예정)

### 1️⃣ 조회 기반 지표

| 지표 | 설명 |
|---|---|
| View Count | 전통주 상세 조회 수 |
| Popular Sool | 많이 조회된 전통주 |
| Trending | 최근 조회 증가율 |

---

### 2️⃣ 리뷰 기반 지표

| 지표 | 설명 |
|---|---|
| 평균 평점 | 품질 인식 |
| 리뷰 수 | 관심도 지표 |
| 평점 분산 | 호불호 판단 |

---

## Phase 3: 추천을 위한 분석 (확장)

### 콘텐츠 기반 추천

- 도수
- 주종
- 지역
- 감각 태그 (향, 단맛, 바디감 등)

> 초기에는 규칙 기반, 이후 ML 적용

---

## 기술 구현 방향

### 데이터 수집
- 서버 로그 기반 조회 기록
- 리뷰 테이블 집계

### 분석 처리
- Python (Pandas)
- SQL Aggregation
- 배치 기반 계산

### 시각화
- Streamlit 대시보드
- 관리자 전용 통계 페이지

---

## 성능 및 운영 고려사항

- 실시간 집계 ❌
- 배치 분석 ⭕
- 통계용 테이블 분리 고려

---

## 분석 문서의 역할

- 기능 추가 기준 제시
- 추천 알고리즘 방향성 정리
- 데이터 직무 역량 증명

---

## 요약

SOOL의 분석 시스템은  
“보여주기 위한 통계”가 아니라  
**서비스 의사결정을 돕는 데이터 구조**를 목표로 설계되었다.
